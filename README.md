# MuSuBi（結び）
### 検索しない音楽体験
### 共感でつながる音楽コミュニティ
---
## 1. このプロジェクトを一言で言うと
**MuSuBiは、
再生数や知名度ではなく、
「人の意思（いいね）」だけで音楽が並ぶWebアプリです。**
音楽を「探す」サービスではなく、
**偶然出会い、共感する体験**を重視しています。
---
## 2. 先に結論（コードを読まなくても分かるポイント）
このプロジェクトで一番伝えたいことは以下です。
* 検索や再生数に依存しない音楽体験を設計したこと
* サービスの思想を、画面・DB設計・ロジックまで一貫させたこと
* あえて実装しなかった機能にも、明確な理由があること
**「なぜこの機能があり、なぜこれは無いのか」**
を説明できることを最優先にしています。
---
## 3. なぜ作ったのか（Why）
私は音楽を専攻しており、
現在の音楽市場に次のような違和感を持っていました。
* 再生数・ランキング至上主義
* ストリーミング数操作などの問題
* 検索前提の受動的な音楽体験
> 音楽は
> 探すものだけではなく、
> 偶然出会い、共感する体験でも良いのではないか
この考えを形にするため、
**MuSuBiではあえて検索機能を実装しない**
という設計判断を行いました。
---
## 4. 検索しない理由（重要）
一般的なWebサービスでは検索は必須ですが、
MuSuBiでは以下を優先しました。
* すでに知っている音楽を探すこと ❌
* 他人の共感から新しい音楽に出会うこと ⭕
その代替として、以下を実装しています。
* 最新投稿一覧
* 共感（いいね）数を基準としたランキング
* ランダム再生機能
これは**機能不足ではなく、思想を優先した設計**です。
---
## 5. サービスの特徴（利用者目線）
### ● 再生数を持たない
* 楽曲再生はYouTube埋め込みを利用
* 再生数はYouTube側で管理
* MuSuBiでは再生数を評価指標にしない
**評価されるのは「いいね」だけ**です。
---
### ● いいね ＝ 意思
* 流し再生では評価されない
* 実際に「良い」と思った人だけが残る
* その集積がランキングになる
> 再生数は結果
> いいねは意思
という考えで設計しています。
---
## 6. 主な機能
* 🎵 音楽投稿
  * YouTube URL共有
  * 音源ファイルアップロード
  * サムネイル自動生成
* ❤️ 共感（いいね）
  * ログインユーザーのみ
  * 誰が共感したかを保持
* 🏆 共感ベースランキング
  * 再生数ではなく共感数基準
  * 「人気」ではなく「推薦」に近い概念
* 💬 コメント機能
  * 楽曲単位で管理
  * 楽曲削除時は連動削除
* 🔐 権限管理
  * USER / ADMIN 分離
  * 削除は管理者のみ
---
## 7. 設計で工夫した点（最小限のコード）
### ① 共感を「数」ではなく「関係」として設計
```java
@ManyToMany
Set<SiteUser> voter;
```
* 何人にいいねされたか、ではなく
* **誰に共感された音楽か**を表現
将来的な推薦・履歴表示も想定した設計です。
---
### ② 思想を反映したランキング取得
```java
SELECT * FROM music
ORDER BY
(SELECT COUNT(*) FROM music_voter WHERE music_id = music.id) DESC
```
データ取得ロジックにも
サービスの思想を反映しています。
---
### ③ 偶然の出会いを作るランダム再生
```java
public Long getRandomMusicId() { ... }
```
検索に頼らず、
「何が流れるか分からない」体験を作るための機能です。
---

## 9. 商業性について

MuSuBiは、
最初から広告収益最大化を目的としたサービスではありません。

ただし、以下のような拡張は想定しています。

* アーティスト支援向け機能
* ポートフォリオ用途
* イベント・ライブ連携
* B2B活用（インディーズ支援）

**後で作ったらいいこと

> 本プロジェクトは学習・ポートフォリオ目的です。
商用化の際は、権利申告・通報・削除フローの実装が必要だと考えています。
検索機能(顧客の利便性のための?)、自分だけの音楽エルボム作り及び自分だけのプレイリスト、
過去に聴取した音楽リスト、音楽をする人のための自分だけのポートフォリオを作ってくれるガイド、
自分の音楽プロフィールを共有できるように、ライブ公演パート。


**「まず体験と設計思想を作る」**
段階として実装しています。
